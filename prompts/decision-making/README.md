# Decision-Making Prompts

**Domain:** Leadership
**Audience:** Engineering leads, CTOs, technical founders

## When to Use

Use these prompts when you face a decision with real tradeoffs and need structured thinking — not just a gut call. They help you evaluate options, surface hidden assumptions, and produce a defensible recommendation.

## Prompts in This Category

| File | Situation |
|------|-----------|
| `technical-stack-decision.md` | Choosing a language, framework, or infrastructure provider |
| `build-vs-buy.md` | Deciding whether to build a capability internally or buy a solution |
| `risk-assessment-ai-feature.md` | Evaluating the risk of shipping an AI-powered feature |
| `prioritization-framework.md` | Prioritizing a backlog of competing technical and product work |

## Tips

- State your constraints before asking for a recommendation — timeline, team size, budget, reversibility.
- Ask for the recommendation AND the strongest counter-argument.
- For irreversible decisions, ask for a "regret minimization" framing.
