# Content Strategy Prompts

**Domain:** Content Creation
**Audience:** Developer audiences, technical decision-makers, general tech enthusiasts

## When to Use

Use these prompts when creating long-form content — articles, courses, documentation, or thought leadership pieces — where structure and depth matter.

## Prompts in This Category

| File | Situation |
|------|-----------|
| `ai-thought-leadership.md` | Writing a thought leadership article on AI engineering |
| `technical-blog-strategy.md` | Planning a technical blog series that builds audience over time |
| `course-content-outline.md` | Outlining a technical course for developer learners |
| `developer-documentation.md` | Writing clear, scannable documentation for a developer tool |

## Tips

- Thought leadership must have a specific, defensible point of view — not just a summary of what's happening.
- Technical blog posts perform best when they solve one specific problem, completely.
- Documentation should be written for someone in a hurry: lead with what they need to do, explain why second.
