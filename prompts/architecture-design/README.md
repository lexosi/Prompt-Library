# Architecture Design Prompts

**Domain:** Technical
**Stack:** Rust, Python, TypeScript, AI Pipelines

## When to Use

Use these prompts at the start of a new system or when an existing system is hitting its limits. They produce concrete architectural decisions — not diagrams that avoid hard choices.

## Prompts in This Category

| File | Situation |
|------|-----------|
| `design-ai-agent-system.md` | Designing a multi-agent orchestration system from scratch |
| `microservices-rust-python.md` | Splitting a monolith into Rust/Python microservices |
| `rag-pipeline-architecture.md` | Designing a Retrieval-Augmented Generation pipeline |
| `game-backend-architecture.md` | Backend architecture for a live-service game |

## Tips

- State your constraints upfront: team size, budget, latency requirements, existing infrastructure.
- If you have an existing system, describe it before asking for an alternative.
- Ask for tradeoffs explicitly — good architecture advice always comes with tradeoffs.
