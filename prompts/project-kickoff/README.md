# Project Kickoff Prompts

**Domain:** Leadership
**Audience:** Engineering teams, clients, stakeholders

## When to Use

Use these prompts at the very start of a project to create alignment documents, kickoff agendas, and scope definitions that prevent misunderstandings and scope creep downstream.

## Prompts in This Category

| File | Situation |
|------|-----------|
| `ai-product-kickoff.md` | Kicking off an AI-powered product with an engineering team |
| `game-feature-kickoff.md` | Starting a new feature for a Fortnite/Roblox experience |
| `client-project-kickoff.md` | Aligning with a client before starting billed work |
| `internal-tool-kickoff.md` | Scoping and starting an internal developer tooling project |

## Tips

- A good kickoff document answers: what are we building, why now, who owns what, and how will we know we're done?
- Include what is explicitly OUT of scope — it prevents the most expensive conversations later.
- Kickoff documents should be one page. If it requires two pages, the scope isn't clear enough yet.
