# Respuesta a Cambio de Alcance

**Categoría:** client-communication
**Cuándo usar:** Rechazar una solicitud fuera del alcance de forma profesional manteniendo la relación intacta.

---

Escribe una respuesta profesional a un cliente que ha solicitado una característica o cambio que está fuera del alcance acordado originalmente. La respuesta debe:

1. Reconocer la solicitud positivamente — muestra que entiendes qué quieren y por qué tiene sentido.
2. Indicar clara (no apologéticamente) que esto cae fuera del acuerdo actual.
3. Indicar el camino a seguir: un nuevo presupuesto, un intercambio de alcance (agregar X, eliminar Y), o un compromiso separado.
4. No ser defensiva ni hacer que el cliente sienta que hizo algo malo.
5. Terminar con un próximo paso claro — una decisión que necesitan tomar, o una llamada para discutir opciones.

Tono: Colaborativo, profesional. Seguro, no rígido.

**Contexto:**
- Lo que el cliente solicitó: [describe]
- Por qué está fuera de alcance: [breve explicación]
- Camino propuesto a seguir: [e.g., "agregar al próximo sprint por $X," "intercambiar por la característica Y," "propuesta separada"]

---

## Ejemplo de output

Hi [Client],

Adding a real-time dashboard makes a lot of sense — I can see why you'd want that visibility into the data as soon as the reports are ready. That's a natural next step.

That said, the current agreement covers the data pipeline and automated PDF reports. A real-time dashboard is a separate frontend component — it's meaningful additional work, so I want to handle it properly rather than squeeze it in.

A few ways we can handle this:

**Option 1:** I put together a separate estimate for the dashboard and we scope it as a follow-on project once the current work ships.

**Option 2:** If you'd like to include it in this engagement, we can swap it for [feature from current scope] and keep the same timeline and budget.

**Option 3:** We deprioritize the dashboard for now and revisit it in 60 days once you've been using the reports.

What works best for you? Happy to jump on a quick call to walk through options if that's easier.

[Name]
