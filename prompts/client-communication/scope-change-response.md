# Scope Change Response

**Category:** client-communication
**When to use:** A client is requesting something outside the agreed scope — you need to respond professionally without damaging the relationship or giving work away for free.

---

Write a professional response to a client who has requested a feature or change that is outside the originally agreed scope. The response must:

1. Acknowledge the request positively — show you understand what they want and why it makes sense.
2. Clearly (not apologetically) note that this falls outside the current agreement.
3. State the path forward: a new estimate, a scope trade (add X, remove Y), or a separate engagement.
4. Not be defensive or make the client feel they did something wrong.
5. End with a clear next step — a decision they need to make, or a call to discuss options.

Tone: Collaborative, professional. Confident, not stiff.

**Context:**
- What the client requested: [describe]
- Why it's out of scope: [brief explanation]
- Proposed path forward: [e.g., "add to next sprint for $X," "swap for feature Y," "separate proposal"]

---

## Example output

Hi [Client],

Adding a real-time dashboard makes a lot of sense — I can see why you'd want that visibility into the data as soon as the reports are ready. That's a natural next step.

That said, the current agreement covers the data pipeline and automated PDF reports. A real-time dashboard is a separate frontend component — it's meaningful additional work, so I want to handle it properly rather than squeeze it in.

A few ways we can handle this:

**Option 1:** I put together a separate estimate for the dashboard and we scope it as a follow-on project once the current work ships.

**Option 2:** If you'd like to include it in this engagement, we can swap it for [feature from current scope] and keep the same timeline and budget.

**Option 3:** We deprioritize the dashboard for now and revisit it in 60 days once you've been using the reports.

What works best for you? Happy to jump on a quick call to walk through options if that's easier.

[Name]
