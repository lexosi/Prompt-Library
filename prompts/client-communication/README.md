# Client Communication Prompts

**Domain:** Business
**Audience:** Technical clients, non-technical stakeholders, project sponsors

## When to Use

Use these prompts when you need to write professional, clear communication to clients — especially in difficult situations like delays, scope changes, or technical explanations.

## Prompts in This Category

| File | Situation |
|------|-----------|
| `explain-technical-delay.md` | Communicating a delay caused by technical complexity |
| `scope-change-response.md` | Responding to a client's request that is outside agreed scope |
| `project-status-update.md` | Writing a weekly or milestone status update |
| `technical-proposal.md` | Proposing a technical approach or solution to a client |

## Tips

- Lead with impact, not technical detail — clients care about timeline and cost first.
- Never blame the technology; own the problem and lead with the solution.
- Keep client messages short. If it requires scrolling, it's too long.
