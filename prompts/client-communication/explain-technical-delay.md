# Explain Technical Delay

**Category:** client-communication
**When to use:** Communicating a delay caused by technical complexity — when you need to be honest, maintain trust, and not lose the client.

---

Write a professional email to a client explaining a project delay caused by a technical issue. The email must:

1. Lead with the impact (new timeline), not the technical explanation.
2. Explain the technical cause in plain language — one short paragraph, no jargon.
3. Be honest about what you didn't know earlier and why you know it now.
4. State specifically what you're doing to address it.
5. End with a clear, specific action — a call, an updated plan, a checkpoint.

Do NOT: apologize excessively, blame the technology, use vague language like "unforeseen complications," or leave the client with an open question.

Tone: Professional, direct, calm. This client is [technical/non-technical].

**Context:**
- Original deadline: [date]
- New deadline: [date]
- Technical cause (plain language): [what went wrong]
- What you're doing about it: [concrete action]
- Client relationship: [new/established, high-stakes/normal]

---

## Example output

Subject: Project timeline update — new delivery date May 15

Hi [Client],

I'm writing to let you know the project delivery is moving from May 1 to May 15. I want to be upfront about what happened and where we stand.

During integration testing this week, we discovered that the third-party payment API we're connecting to returns data in a different format than their documentation specifies. This required us to rebuild part of the data processing layer we had already completed. We couldn't have caught this earlier — the discrepancy only appears when processing real transaction data, which we couldn't access until this stage.

We have the fix underway. The rebuilt layer will be complete by May 8, giving us the remaining week for testing and delivery.

Can we schedule a 20-minute call on Thursday to walk you through the updated plan? I'll send the detailed timeline before the call so you can review it in advance.

[Name]
